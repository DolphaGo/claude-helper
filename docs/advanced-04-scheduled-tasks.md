## 🎯 Scheduled Tasks란?

**Scheduled Tasks**는 정해진 시간에 자동으로 Claude에게 프롬프트를 보내는 기능입니다.

### 실생활 비유

스마트폰의 알람과 비슷합니다:
- 매일 오전 9시: "오늘 할 일 정리"
- 매주 금요일: "주간 리포트 작성"
- 매시간: "에러 로그 확인"

### Cron Channel과의 차이

```
Cron Channel (알림만)
┌─────────────────────────────────────┐
│  매일 오전 9시                       │
└─────────────────────────────────────┘
         ↓ 트리거
┌─────────────────────────────────────┐
│  Claude Code                        │
│  "Git 상태를 확인하세요" 메시지 표시│  ← 알림만
└─────────────────────────────────────┘
         ↓ 사용자가 수동으로 확인


Scheduled Tasks (자동 실행)
┌─────────────────────────────────────┐
│  매일 오전 9시                       │
└─────────────────────────────────────┘
         ↓ 트리거
┌─────────────────────────────────────┐
│  Claude Code                        │
│  1. git status 실행                 │  ← 자동 실행
│  2. 커밋 분석                       │
│  3. 리포트 작성                     │
│  4. 결과 표시                       │
└─────────────────────────────────────┘
         ↓ 완료된 작업 확인
```

| 비교 | Cron Channel | Scheduled Tasks |
|------|--------------|-----------------|
| 기능 | 알림만 표시 | Claude가 작업 수행 |
| 용도 | 상기시키기 | 자동으로 작업 |
| 예시 | "확인하세요" 메시지 | 실제로 확인하고 리포트 작성 |

**실전 예시:**

Cron Channel:
```
9:00 AM → 💬 "Git 커밋을 확인하세요"
         (사용자가 /my-plugin:gs 실행 필요)
```

Scheduled Tasks:
```
9:00 AM → 🤖 자동으로 Git 분석 실행
         → 📊 "최근 5개 커밋 분석 완료..." 리포트 생성
```

---

## ⚙️ 기본 사용법

### CLI에서 생성

```bash
claude cron create "0 9 * * *" "오늘의 Git 커밋을 요약해주세요"
```

**구성 요소:**
- `0 9 * * *`: Cron 표현식 (매일 오전 9시)
- `"프롬프트"`: Claude에게 전달할 메시지

### 대화 중 생성

Claude Code 대화 중에도 생성 가능합니다:

```
매일 오전 9시에 Git 커밋 요약을 자동으로 해줘
```

Claude가 자동으로 Scheduled Task를 생성합니다.

## ⏰ Cron 표현식

Cron 표현식은 실행 시간을 지정합니다.

### 기본 형식

```
분 시 일 월 요일
*  *  *  *  *
```

### 자주 사용하는 패턴

| 표현식 | 의미 |
|--------|------|
| `0 9 * * *` | 매일 오전 9시 |
| `0 */2 * * *` | 2시간마다 |
| `*/30 * * * *` | 30분마다 |
| `0 9 * * 1` | 매주 월요일 오전 9시 |
| `0 9 1 * *` | 매월 1일 오전 9시 |
| `0 9 * * 1-5` | 평일 매일 오전 9시 |

### 예제 해석

```bash
# 매일 오전 9시 30분
claude cron create "30 9 * * *" "..."

# 평일 오후 6시
claude cron create "0 18 * * 1-5" "..."

# 매주 금요일 오후 3시
claude cron create "0 15 * * 5" "..."
```

## 💡 실용적인 예제

### 예제 1: 매일 Git 상태 요약

```bash
claude cron create "0 9 * * *" "어제부터 오늘까지의 Git 커밋을 요약하고, 진행 중인 작업을 정리해주세요"
```

**효과:**
매일 오전 9시에 자동으로:
- 최근 커밋 내역 확인
- 진행 중인 작업 정리
- 오늘 할 일 제안

### 예제 2: 주간 코드 리뷰

```bash
claude cron create "0 10 * * 1" "지난 주의 코드 변경사항을 리뷰하고, 개선이 필요한 부분을 찾아주세요"
```

**효과:**
매주 월요일 오전 10시에:
- 지난 주 커밋 분석
- 코드 품질 확인
- 개선 포인트 제안

### 예제 3: 정기 테스트 실행

```bash
claude cron create "0 */3 * * *" "전체 테스트를 실행하고, 실패한 테스트가 있으면 원인을 분석해주세요"
```

**효과:**
3시간마다:
- 테스트 실행
- 실패 시 원인 분석
- 수정 방안 제안

### 예제 4: 의존성 업데이트 확인

```bash
claude cron create "0 9 * * 1" "package.json의 의존성을 확인하고, 업데이트가 필요한 패키지를 알려주세요"
```

**효과:**
매주 월요일 오전 9시에:
- 오래된 패키지 확인
- 보안 취약점 체크
- 업데이트 우선순위 제안

## 📋 Task 관리

### Task 목록 보기

```bash
claude cron list
```

**출력 예시:**

```
ID: 1
Schedule: 0 9 * * *
Prompt: Git 커밋 요약
Next run: 2024-03-20 09:00

ID: 2
Schedule: 0 18 * * 5
Prompt: 주간 리포트
Next run: 2024-03-22 18:00
```

### Task 삭제

```bash
claude cron delete 1
```

ID 1번 task를 삭제합니다.

### 모든 Task 삭제

```bash
claude cron delete --all
```

## 🎨 고급 사용법

### 프로젝트별 설정

프로젝트 디렉토리에서 실행하면 해당 프로젝트에서만 작동합니다.

```bash
cd /path/to/my-project
claude cron create "0 9 * * *" "이 프로젝트의 TODO 주석을 찾아서 정리해주세요"
```

### 복잡한 프롬프트

여러 줄의 프롬프트도 가능합니다:

```bash
claude cron create "0 9 * * 1" "
주간 리포트를 작성해주세요:
1. 지난 주 완료된 기능
2. 새로 추가된 버그
3. 이번 주 계획
4. 팀원들에게 공유할 내용
"
```

### 조건부 실행

프롬프트에 조건을 포함할 수 있습니다:

```bash
claude cron create "0 9 * * *" "Git 상태를 확인하고, 커밋되지 않은 변경사항이 있으면 알려주세요"
```

## 🎓 실전 패턴

### 패턴 1: 아침 대시보드

```bash
claude cron create "0 9 * * 1-5" "
오늘의 개발 대시보드:
1. Git 브랜치 상태
2. 미완료 TODO 주석 개수
3. 실패한 테스트 여부
4. 최근 에러 로그
"
```

**효과:**
평일 아침마다 프로젝트 상태를 한눈에 확인합니다.

### 패턴 2: 금요일 정리

```bash
claude cron create "0 17 * * 5" "
주말 전 정리:
1. 모든 브랜치를 main과 비교
2. 머지되지 않은 변경사항 확인
3. 다음 주 우선순위 제안
"
```

**효과:**
주말 전에 작업 상태를 정리하고 다음 주를 준비합니다.

### 패턴 3: 야간 분석

```bash
claude cron create "0 2 * * *" "
야간 코드 분석:
1. 전체 코드에서 TODO, FIXME, HACK 주석 찾기
2. 복잡도가 높은 함수 찾기
3. 중복 코드 탐지
"
```

**효과:**
야간에 자동으로 코드 품질을 분석합니다.

### 패턴 4: 정기 백업 확인

```bash
claude cron create "0 10 * * *" "
백업 상태 확인:
1. 최근 커밋이 원격에 푸시되었는지 확인
2. 중요한 변경사항이 로컬에만 있는지 체크
3. 백업이 필요하면 알림
"
```

## 🛠️ Skill과 연동

Scheduled Task에서 Skill을 호출할 수 있습니다.

### 예제: 정기 상태 확인

**Task 생성:**

```bash
claude cron create "0 9 * * *" "/my-first-plugin:gs"
```

매일 오전 9시에 Git 상태 skill을 실행합니다.

### 예제: 자동 테스트

**Skill 생성:**

```bash
mkdir -p ~/.claude/skills/auto-test
cat > ~/.claude/skills/auto-test/SKILL.md << 'SKILLEOF'
---
description: 자동으로 테스트를 실행하고 결과를 분석합니다
---

# Auto Test

## 1. 테스트 실행
전체 테스트 스위트를 실행하세요.

## 2. 결과 분석
- 통과: 개수만 표시
- 실패: 상세 분석

## 3. 실패 시 조치
실패한 테스트의 원인을 파악하고 수정 방안을 제안하세요.
SKILLEOF
```

**Task 생성:**

```bash
claude cron create "0 */6 * * *" "/auto-test"
```

6시간마다 자동으로 테스트를 실행하고 분석합니다.

## 📚 핵심 정리

### 명령어 요약

```bash
# Task 생성
claude cron create "cron표현식" "프롬프트"

# Task 목록
claude cron list

# Task 삭제
claude cron delete <ID>

# 모든 Task 삭제
claude cron delete --all
```

### Cron 표현식 치트시트

```
# 매일
0 9 * * *           # 오전 9시

# 주기적
*/30 * * * *        # 30분마다
0 */2 * * *         # 2시간마다

# 특정 요일
0 9 * * 1           # 월요일 오전 9시
0 9 * * 1-5         # 평일 오전 9시
0 9 * * 5           # 금요일 오전 9시

# 특정 날짜
0 9 1 * *           # 매월 1일 오전 9시
0 9 15 * *          # 매월 15일 오전 9시
```

### 사용 시나리오

**매일 반복 작업:**
- 코드 리뷰
- 테스트 실행
- 로그 확인

**주간 반복 작업:**
- 주간 리포트
- 의존성 업데이트 확인
- 코드 정리

**정기 점검:**
- 보안 스캔
- 성능 분석
- 백업 확인

## ⚠️ 주의사항

### 실행 시간

Task는 Claude Code가 실행 중일 때만 작동합니다.
- 컴퓨터가 꺼져 있으면 실행 안 됨
- 놓친 Task는 다음 실행 시간에 실행

### 작업 시간

너무 오래 걸리는 작업은 피하세요:
- ✅ 상태 확인, 요약
- ❌ 전체 빌드, 배포

### 프롬프트 작성

명확하고 구체적으로:
- ✅ "Git 커밋을 요약하고 TODO를 찾아줘"
- ❌ "뭔가 해줘"

### 중복 방지

비슷한 Task가 겹치지 않도록:

```bash
# 나쁜 예
claude cron create "0 9 * * *" "Git 상태 확인"
claude cron create "5 9 * * *" "Git 커밋 확인"  # 5분 후 중복

# 좋은 예
claude cron create "0 9 * * *" "Git 상태와 커밋 확인"
```

## 🔍 디버깅

### Task가 실행되지 않을 때

**1. Task 목록 확인:**

```bash
claude cron list
```

**2. 다음 실행 시간 확인:**

```
Next run: 2024-03-20 09:00
```

현재 시간과 비교하세요.

**3. Cron 표현식 테스트:**

온라인 도구 사용:
- [Crontab Guru](https://crontab.guru)

**4. Claude Code 실행 확인:**

Task 실행 시간에 Claude Code가 실행 중이어야 합니다.
